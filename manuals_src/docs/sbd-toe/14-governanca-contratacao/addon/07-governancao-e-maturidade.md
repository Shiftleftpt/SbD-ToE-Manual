# 📈 Governança e Maturidade Organizacional {governanca-contratacao:addon:governancao-maturidade}

Este anexo descreve como o Capítulo 14 contribui para a **maturidade de segurança** da organização, com base nos modelos SAMM, BSIMM, SSDF e SLSA, e como essas práticas se traduzem em controlo efetivo.

---

## 🔢 1. Alinhamento com frameworks de maturidade {governanca-contratacao:addon:governancao-maturidade#1_alinhamento_com_frameworks_de_maturidade}

| Framework | Domínio                         | Práticas relacionadas SbD-ToE                      | Nível sugerido |
| --------- | ------------------------------- | -------------------------------------------------- | -------------- |
| SAMM v2.1 | Governance > Strategy & Metrics | Modelo de decisão de risco, exceções, KPIs         | 2 / 3          |
| SAMM v2.1 | Governance > Supplier Security  | Questionários, contratos, validação, onboarding    | 2 / 3          |
| BSIMM13   | SM1.1 / SR1.3 / SR2.x           | Governaça institucional, owners, exceções          | 2 / 3          |
| NIST SSDF | PO.1 / PO.3 / RV.2              | Políticas, roles, aprovações, revisões contratuais | 2 / 3          |
| SLSA v1.0 | Provenance / Compliance         | Controlo de cadeias externas e rastreabilidade     | 1 / 4          |

> 📊 As práticas de governaça permitem escalar o SbD-ToE com consistência organizacional.

---

## 🔄 2. Escalonamento de maturidade {governanca-contratacao:addon:governancao-maturidade#2_escalonamento_de_maturidade}

| Nível | Característica-chave                                                           |
| ----- | ------------------------------------------------------------------------------ |
| 1     | Aplicação pontual de requisitos, sem rastreabilidade ou validação formal       |
| 2     | Modelo de exceção definido, requisitos mapeados a risco, contratos atualizados |
| 3     | Rastreabilidade completa, owners designados, validação recorrente automatizada |

> 🔹 O Cap. 14 fornece os elementos organizacionais para atingir o Nível 3 de maturidade.

---

## 🌍 3. Aplicação transversal {governanca-contratacao:addon:governancao-maturidade#3_aplicacao_transversal}

A maturidade de governaça afeta **todos os restantes capítulos**:

* Define quando e como aplicar requisitos (Cap. 2);
* Impede exceções não rastreadas (Cap. 1);
* Controla qualidade de integração e terceiros (Cap. 5, 7, 9);
* Estabelece mecanismos formais de validação (Cap. 10, 11, 12).

---

## 🗓️ 4. Maturidade como programa evolutivo {governanca-contratacao:addon:governancao-maturidade#4_maturidade_como_programa_evolutivo}

A evolução do modelo SbD-ToE pode ser guiada por:

* Adoção formal do modelo de governaça (Cap. 14.1);
* Integração de revisões e exceções num ciclo trimestral;
* Geração de KPIs e dashboards (Cap. 14.30);
* Auditorias cruzadas entre aplicações, contratos e rastreabilidade.

---

## ✅ Recomendações finais {governanca-contratacao:addon:governancao-maturidade#recomendacoes_finais}

* Rever anualmente a estrutura de governaça e roles atribuídos;
* Consolidar dados operacionais (checklists) com dados de processo (decisões);
* Monitorizar a evolução de maturidade usando frameworks externos como baseline.
