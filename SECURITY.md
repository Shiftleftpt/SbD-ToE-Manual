# Política de Segurança - SbD-ToE

O projeto **Security by Design - Theory of Everything (SbD-ToE)** segue práticas de divulgação responsável.

## 🔎 Como reportar vulnerabilidades

1. **Não** abrir um issue público.
2. Enviar e-mail para **security@shiftleft.pt** com:
   - Descrição do problema e impacto
   - Passos para reproduzir (se possível)
   - Ficheiros ou secções afetadas

Receberás resposta em até **72 horas**.

## 🧩 âmbito

- Código de build, workflows GitHub e scripts auxiliares.
- Conteúdos Markdown e pipelines de geração de site.

## 🕒 Prazos de resposta

| Etapa | Prazo máximo |
|-------|---------------|
| Triagem inicial | 3 dias úteis |
| Mitigação / plano | 14 dias |
| Divulgação pública | 30 dias ou após correção |
